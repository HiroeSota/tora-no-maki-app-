## HTMLクイズデータの更新規約

### ファクトチェック（作成前・必須）
- クイズを新規作成・修正する前に、必ず**自律的にファクトチェックを2回**行うこと。
  - 1回目：問題文・選択肢・正解が正しいか自分の知識で検証する
  - 2回目：1回目とは別の切り口（例：反例がないか・他の選択肢が正解になり得ないか）で再検証する
- **確証が持てない問題は作らない。** 曖昧・うろ覚えの情報をそのまま問題にしてはならない。
- 以下の条件をすべて満たしているか確認すること：
  1. 生成した情報が客観的に正しいか
  2. 正解の選択肢が唯一の正解になっているか（他の選択肢も正解に見える曖昧な表現はNG）
  3. hint の文章に誤字脱字や論理の破綻がないか

### クイズデータの構造ルール（厳守）
新規問題オブジェクトは必ず以下の形式に従うこと：

```js
{
  category: 'カテゴリ名',  // 既存カテゴリから選ぶ
  text: '問題文',
  choices: ['A', 'B', 'C', 'D'],  // 必ず4つ
  answer: 0,                       // 0〜3 の整数（choices の正解インデックス）
  difficulty: 1, isNew: true,      // difficulty は 1・2・3 のいずれか。isNew は必須
  hint: '補足説明',                // 必須。「解説」という文字は使わない
}
```

- `choices` は**必ず4つ**。3つ以下・5つ以上は禁止。
- `answer` は **0〜3 の整数**のみ。文字列・範囲外の数値は禁止。
- `difficulty` は **1・2・3** のいずれか。
- `isNew: true` は**必須**。新規追加問題には必ず付ける。「🆕 新着問題」フィルターで自動表示される仕組み。
- `hint` は**必須**。省略・空文字は禁止。
- **「解説」という文字をファイル内に書いてはならない**（hint フィールドで代替する）。

### 単語リストの同時更新（必須）
問題を新規作成したら、その問題に登場する新しい用語を**必ず同時に**単語リストへ追加すること。追加漏れは表示バグの原因になる。

単語リストへの追加は以下の **3箇所すべて** に行うこと：

1. **`GLOSSARY`**（定義辞書）— 用語の説明文を追加。**これが抜けると単語リストに表示されない。**
2. **`GLOSSARY_LEVEL`** — レベルを `1`（基本）・`2`（中級）・`3`（応用）で指定。
3. **`WORD_LIST_GROUPS`** — 対応するカテゴリの `terms` 配列に用語名を追加。

### Stop フック
- `Stop` フックが実行された際、以下をすべて確認し、1つでも不備があれば完了報告をせず即座に修正して再実行すること：
  - ファクトチェックが十分か
  - 問題の構造ルールを満たしているか（choices 4つ・answer 0〜3・hint 必須など）
  - 新しい用語の単語リスト3箇所（`GLOSSARY`・`GLOSSARY_LEVEL`・`WORD_LIST_GROUPS`）への追加漏れがないか
  - 新規問題に `isNew: true` が付いているか
